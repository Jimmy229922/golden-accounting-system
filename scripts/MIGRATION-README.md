# تحديث أرقام الفواتير والسندات

## 📝 النظام الجديد

تم تحديث نظام الترقيم ليكون موحداً مع بادئة حرفين لكل نوع:

- **فواتير المبيعات**: `SL-0001`, `SL-0002`, ...
- **فواتير المشتريات**: `PC-0001`, `PC-0002`, ...
- **مردودات المبيعات**: `SR-0001`, `SR-0002`, ...
- **مردودات المشتريات**: `PR-0001`, `PR-0002`, ...
- **سندات التحصيل**: `RC-0001`, `RC-0002`, ...
- **سندات السداد**: `PY-0001`, `PY-0002`, ...

## 🚀 كيفية التحديث

### 1. عمل نسخة احتياطية

**مهم جداً**: قبل تشغيل التحديث، قم بعمل نسخة احتياطية من قاعدة البيانات:

```powershell
# انسخ قاعدة البيانات
Copy-Item "$env:USERPROFILE\accounting-system-data\accounting.db" "$env:USERPROFILE\accounting-system-data\accounting-backup-$(Get-Date -Format 'yyyyMMdd-HHmmss').db"
```

### 2. تشغيل سكريبت التحديث

من مجلد المشروع الرئيسي:

```powershell
node scripts/migrate-invoice-numbers.js
```

### 3. التحقق من النتائج

سيعرض السكريبت تقريراً بجميع التغييرات:

```
✅ Migration completed successfully!

📊 Summary:
   - Sales Invoices: SL-XXXX
   - Purchase Invoices: PC-XXXX
   - Sales Returns: SR-XXXX
   - Purchase Returns: PR-XXXX
   - Receipts: RC-XXXX
   - Payments: PY-XXXX
```

## ⚠️ ملاحظات مهمة

1. **تأكد من إغلاق التطبيق** قبل تشغيل السكريبت
2. **لا تشغل السكريبت أكثر من مرة** على نفس قاعدة البيانات
3. **احفظ النسخة الاحتياطية** في مكان آمن
4. إذا حدث خطأ، استعد النسخة الاحتياطية:

```powershell
Copy-Item "$env:USERPROFILE\accounting-system-data\accounting-backup-XXXXXX.db" "$env:USERPROFILE\accounting-system-data\accounting.db"
```

## 🔄 التأثير على النظام

- ✅ جميع الفواتير والسندات القديمة ستحصل على أرقام جديدة
- ✅ الترتيب الزمني للعمليات سيبقى كما هو
- ✅ جميع العلاقات بين الجداول ستبقى سليمة
- ✅ الفواتير والسندات الجديدة ستستخدم النظام الجديد تلقائياً

## 🆘 في حالة المشاكل

إذا واجهت أي مشكلة:

1. أوقف التطبيق فوراً
2. استعد النسخة الاحتياطية
3. تواصل مع الدعم الفني
